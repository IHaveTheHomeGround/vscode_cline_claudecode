# Risk_Matrix.md — v3.0

## 판정 플로우

```
새 작업
  │
  ▼
[Q1] 되돌릴 수 없는 작업? (삭제, 외부 전송, DB 수정)
  │
  ├─ YES → CRITICAL
  │
  └─ NO
       │
       ▼
     [Q2] 설정·환경파일 수정? (.env, config, 시스템 설정)
       │
       ├─ YES → CRITICAL
       │
       └─ NO
            │
            ▼
          [Q3] 외부 API / 서비스 호출 포함?
            │
            ├─ YES → CRITICAL
            │
            └─ NO
                 │
                 ▼
               [Q4] 복수 파일 수정 OR 새 파일 생성 OR 10 LOC 초과?
                 │
                 ├─ YES → STANDARD
                 │
                 └─ NO → LIGHTWEIGHT
```

**원칙**: 경계 케이스는 상위 등급으로 판정.

---

## 등급별 정의

### 🟢 LIGHTWEIGHT
**조건** (모두 충족):
- 단일 파일
- 10 LOC 이하 변경
- 되돌리기 쉬움
- 다른 파일/시스템 영향 없음

**처리**: 자율 실행 → 완료 보고  
**백업**: 선택사항

**작업 유형별 예시**:
| 유형 | 예시 |
|------|------|
| CODE | 오타 수정, 변수명 변경 1개 |
| DOC | 단락 수정, 제목 변경 |
| ANALYSIS | 파일 구조 확인 (읽기만) |
| SCRIPT | echo 출력 스크립트 수정 |

---

### 🟡 STANDARD
**조건** (하나라도 해당):
- 복수 파일 수정
- 새 파일/디렉터리 생성
- 10 LOC 초과
- 기존 로직 변경 (리팩토링)

**처리**: 자율 실행 → 완료 후 상세 보고  
**백업**: 필수 (기존 파일 수정 시)

**작업 유형별 예시**:
| 유형 | 예시 |
|------|------|
| CODE | 새 함수/모듈 추가, 리팩토링 |
| DOC | 새 문서 파일 생성, 복수 파일 정리 |
| ANALYSIS | 데이터 처리 후 결과 파일 생성 |
| SCRIPT | 새 자동화 스크립트 작성 |

---

### 🔴 CRITICAL
**조건** (하나라도 해당):
- 파일/디렉터리 삭제
- .env, config, 시스템 설정 수정
- 외부 API / 서비스 호출
- 데이터베이스 수정
- 배포 / 빌드 스크립트 실행
- 대규모 일괄 수정 (10개 파일 이상)

**처리**: 계획 전체 제시 → 사용자 승인 → 실행  
**백업**: 필수

**작업 유형별 예시**:
| 유형 | 예시 |
|------|------|
| CODE | 핵심 비즈니스 로직 변경, 의존성 대규모 교체 |
| DOC | 중요 문서 덮어쓰기, 폴더 구조 재편 |
| ANALYSIS | 원본 데이터 변환/삭제 포함 |
| SCRIPT | 실제 배포, 파일 대량 이동/삭제 |
